# Fix Sidebar Horizontal Scroll Implementation Plan

## Overview
Fix the issue where sidebar task items cause horizontal overflow and content clipping.

## Current State Analysis
- `index.css:21`: `.task-sidebar` has fixed padding and `overflow: hidden`.
- `index.css:33`: `.sidebar-list` has `overflow-y: auto` and `padding-right: 10px`.
- `TaskSidebar.jsx`: Draggable items have `width: 100%` behavior within a flex container that might be slightly too wide due to internal padding/borders.

## Implementation Approach
1. Enforce `overflow-x: hidden` on the sidebar list container.
2. Adjust padding to prevent child elements from being pushed out by the vertical scrollbar.
3. Clean up redundant styles in `TaskSidebar.jsx`.

## Phase 1: CSS Sanitization
### Overview
Update `index.css` to prevent horizontal overflow.

### Changes Required:
#### 1. `src/index.css`
**Changes**: Update `.sidebar-list` to include `overflow-x: hidden`.

```css
.sidebar-list {
  flex-grow: 1;
  overflow-y: auto;
  overflow-x: hidden; /* KILL HORIZONTAL SCROLL */
  margin-top: 20px;
  padding-right: 5px; /* Reduced to avoid scrollbar overlap */
  min-height: 100px;
}
```

## Phase 2: Component Cleanup
### Overview
Remove redundant styles and ensure layout stability in `TaskSidebar.jsx`.

### Changes Required:
#### 1. `src/TaskSidebar.jsx`
**Changes**: Remove redundant `style` object from the `Droppable` container.

```jsx
<div 
  className="sidebar-list" 
  {...provided.droppableProps} 
  ref={provided.innerRef} 
  // REMOVE style={{ flexGrow: 1, overflowY: 'auto' }}
>
```

### Success Criteria:
#### Manual:
- [ ] Sidebar content fits perfectly.
- [ ] No horizontal scrollbar appears when task names are long or list is full.
- [ ] Borders of task items are not clipped.
