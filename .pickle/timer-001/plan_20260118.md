# Mobile Timer Customization Implementation Plan

## Overview
Add preset selection functionality to the mobile header version of the Pomodoro timer.

## Current State Analysis
- `src/PomodoroTimer.jsx`: The `header` variant is a simple horizontal row with time, play/pause, and reset.
- Presets are only available in the `full` variant (sidebar tool).

## Implementation Approach
1. Add a small `<select>` dropdown or a set of clickable spans for presets next to the timer display in the `header` variant.
2. Use a `<select>` for maximum space efficiency given the narrow mobile header.
3. Style it to look like a "Paper" artifact.

## Phase 1: Header Variant Refactor
### Overview
Inject the preset selector into the `header` return block.

### Changes Required:
#### 1. `src/PomodoroTimer.jsx`
**Changes**: Add a `select` element between the time display and the toggle button.

```jsx
<select 
  value={selectedPreset} 
  onChange={(e) => setSelectedPreset(Number(e.target.value))}
  style={{
    background: 'none',
    border: '1px solid black',
    borderRadius: '3px',
    fontFamily: 'inherit',
    fontSize: '0.8em',
    padding: '2px'
  }}
>
  {PRESETS.map(p => (
    <option key={p} value={p}>{p}m</option>
  ))}
</select>
```

### Success Criteria:
#### Manual:
- [ ] On mobile view, a dropdown appears next to the timer.
- [ ] Changing the dropdown value updates the `timeLeft` and resets the `isActive` state (inherited from `useEffect`).
- [ ] The dropdown value correctly reflects the current `selectedPreset`.
