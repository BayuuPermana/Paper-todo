# Container Merge Implementation Plan

## Overview
Remove redundant nesting of the `TaskSidebar` container. Merge the outer wrapper in `App.jsx` with the component's internal container.

## Current State Analysis
- `App.jsx` has a `div` with `className="task-sidebar"` wrapping `TaskSidebar`.
- `TaskSidebar.jsx` has a `div` with `className="task-sidebar"` as its root.
- Result: Double borders, double padding, double trouble.

## Implementation Approach
1. Refactor `TaskSidebar.jsx` to accept `className` as a prop.
2. Combine the passed `className` with its own `task-sidebar` class.
3. Update `App.jsx` to remove the redundant `div` and pass the mobile visibility classes directly to `TaskSidebar`.

## Phase 1: TaskSidebar Component Update
### Overview
Modify the component to handle external classes.

### Changes Required:
#### 1. `src/TaskSidebar.jsx`
**Changes**: Add `className` to props and apply it to the root `div`.

```jsx
function TaskSidebar({ todos, selectedTodoId, onSelectTodo, onAddTodo, className }) {
  // ...
  return (
    <div className={`task-sidebar ${className || ''}`} style={{ ... }}>
      {/* ... */}
    </div>
  );
}
```

## Phase 2: App.jsx Integration
### Overview
Remove the redundant wrapper.

### Changes Required:
#### 1. `src/App.jsx`
**Changes**: Remove the outer `div` and pass classes to `TaskSidebar`.

```jsx
<DragDropContext onDragEnd={handleDragEnd} key={activeTab}>
  <TaskSidebar
    todos={todos}
    selectedTodoId={selectedTodoId}
    onSelectTodo={setSelectedTodoId}
    onAddTodo={addTodo}
    className={activeTab === 'archive' ? 'mobile-show' : 'mobile-hide'}
  />
</DragDropContext>
```

## Success Criteria:
#### Manual:
- [ ] Archive rack has a single border.
- [ ] Mobile visibility still works (Sidebar toggles correctly on mobile).
- [ ] Drag and drop functionality is unaffected.
