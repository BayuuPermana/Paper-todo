**Date**: 2026-01-19

## 1. Executive Summary

## 3. Findings & Analysis
- **The "Whack" Factor**: The emoji buttons in the header are small, lack proper affordance (no border/background indicating "clickable" state until clicked or active), and rely on OS-specific emoji rendering which varies wildly.
- **Theme Clash**: The app uses a "Paper/Scribble" aesthetic (e.g., `scribble.svg`, font choices). Emojis are colorful and digital, breaking the immersion.
- **Constraint**: We need to keep the header compact (`height: 60px` in CSS).

## 4. Technical Constraints
- **Height**: Header is fixed at 60px. Button height must fit within ~40-50px.
- **Dependencies**: No external icon libraries (FontAwesome, etc.) are currently installed/used. We should use inline SVGs or CSS shapes to maintain "God Mode" independence.

## 5. Architecture Documentation
- **Styles**: `src/index.css` contains global styles.
- **Assets**: `src/assets/scribble.svg` exists and can be referenced for style matching (rough lines).
