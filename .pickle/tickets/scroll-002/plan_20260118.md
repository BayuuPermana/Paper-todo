# Graphite-Style Scrollbar Implementation Plan

## Overview
Implement custom CSS styles for scrollbars to match the application's "Paper" aesthetic.

## Current State Analysis
- `src/index.css`: No scrollbar overrides. `.left-segment` has `overflow-y: auto`.

## Implementation Approach
1. **Define Global Scrollbar Styles**: Add WebKit and standard scrollbar rules to `src/index.css`.
2. **Aesthetic Refinement**: Use a "graphite" color palette and thin width.
3. **Application**: Apply to the root and specific containers like `.left-segment`.

## Phase 1: CSS Implementation
### Overview
Add the custom scrollbar rules.

### Changes Required:
#### 1. `src/index.css`
**Changes**: Add scrollbar pseudo-elements.
```css
/* Custom Scrollbar for WebKit */
::-webkit-scrollbar {
  width: 8px;
}

::-webkit-scrollbar-track {
  background: transparent;
  border-left: 1px dashed #eee; /* Faint pencil guide line */
}

::-webkit-scrollbar-thumb {
  background: #888; /* Graphite grey */
  border: 2px solid #fff; /* Padding effect */
  border-radius: 10px;
}

::-webkit-scrollbar-thumb:hover {
  background: #555; /* Darker graphite on hover */
}

/* Firefox support */
* {
  scrollbar-width: thin;
  scrollbar-color: #888 transparent;
}
```

### Success Criteria:
#### Automated:

- [ ] **Hover Check**: Verify the color changes when hovering over the thumb.
