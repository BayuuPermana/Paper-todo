# Research: Mobile Neural Hub Architecture

**Date**: 2026-01-18

## 1. Executive Summary
The current mobile layout is a linear vertical stack that hides tools and forces excessive navigation. We will refactor to a tabbed architecture with a persistent Command Header. This requires a new layer of state management for view switching and a visual overhaul of the tool layout for small viewports.

## 2. Technical Context
- **State Source**: `src/App.jsx`.
- **CSS File**: `src/index.css`.
- **Existing Breakpoint**: 1200px (Desktop) vs < 1200px (Mobile/Tablet).
- **Tools**: `PomodoroTimer` and `PaperCalendar` currently live in the `right-segment`.

## 3. Findings & Analysis
- **Tab State**: A single string state `activeTab` is sufficient to control visibility.
- **Sticky Layout**: 
    - Header: `position: fixed; top: 0; height: ~60px`.
    - Tab Bar: `position: fixed; bottom: 0; height: ~60px`.
    - Content: `padding: 70px 0; overflow-y: auto`.
- **Tool Condensation**: On mobile, the `right-segment` will disappear. The Timer and a "Mini-Date" string will move to the sticky header.
- **Aesthetic Integration**: Use dashed borders and paper textures for the new mobile bars to maintain the "desk" feel.

## 4. Technical Constraints
- **Z-Index**: The 'Crumple' animation and sidebar hover previews must stay below the fixed bars.
- **Orientation**: Ensure the `input-matrix` remains horizontal as requested.

## 5. Architecture Documentation
- State: `activeTab` ('archive', 'workspace').
- Visibility Classes: `.mobile-hide`, `.mobile-visible`.
