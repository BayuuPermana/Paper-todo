# Research: Custom Scrollbar Pseudo-elements

**Date**: 2026-01-18

## 1. Executive Summary
Standard OS scrollbars are disruptive to the "Paper" UI. By using WebKit pseudo-elements, we can style the scrollbar to mimic pencil or graphite strokes, ensuring visual consistency across all scrollable segments.

## 2. Technical Context
- **Target Container**: `.left-segment` (Lines 38-42).
- **Current Scroll**: Browser default.
- **Color Palette**: 
    - Background: `#fff` (paper)
    - Ink: `black`
    - Graphite: `#888` / `#aaa`

## 3. Findings & Analysis
- **Pseudo-elements**:
    - `::-webkit-scrollbar`: Main scrollbar width/height.
    - `::-webkit-scrollbar-track`: The path the thumb travels.
    - `::-webkit-scrollbar-thumb`: The draggable part.
- **Aesthetic Mapping**:
    - Thumb: Dark grey (`#555`), rounded ends (`10px`), thin width (`6px`).
    - Track: Faint grey line (`#eee`) or transparent.
- **Standard CSS**: `scrollbar-color` and `scrollbar-width` for Firefox.
- **Scrollbar Gutter**: `scrollbar-gutter: stable` can prevent layout shifts when content expands.

## 4. Technical Constraints
- **Specificity**: Global styles in `index.css` are sufficient.
- **Browser Compatibility**: Primarily targeting Chrome/Edge/Safari (WebKit). Firefox needs `scrollbar-color`.

## 5. Architecture Documentation
- Current: Default OS behavior.
- Required: Global CSS overrides for scrollable components.
