# Vertical Margins Implementation Plan

## Overview
Implement the "Breathing Room Protocol" by switching the CSS centering strategy. We will replace the rigid `align-items: center` on the body with a flexible `margin: auto` approach on the container, combined with body padding. This ensures vertical centering for short lists and accessible scrolling for long lists, with guaranteed margins.

## Current State Analysis
- `src/index.css`:
    - `body` uses `align-items: center`.
    - `.paper-container` has no vertical margin.

## Implementation Approach
1.  **Refactor Body Layout**: Switch `align-items` to `flex-start` to prevent top-clipping. Add vertical padding.
2.  **Refactor Container Layout**: Use `margin: auto` to achieve safe vertical centering within the flex parent.

## Phase 1: CSS Refactor
### Overview
Apply the new spacing and alignment rules.

### Changes Required:
#### 1. `src/index.css`
**Changes**: Update `body` and `.paper-container` rules.
```css
body {
  margin: 0;
  display: flex;
  justify-content: center;
  align-items: flex-start; /* Changed from center to prevent clipping */
  min-height: 100vh;
  background-color: #f0f2f5;
  font-family: 'Comic Sans MS', 'Chalkboard SE', 'Comic Neue', sans-serif;
  color: #333;
  padding: 50px 0; /* Add breathing room */
  box-sizing: border-box; /* Ensure padding is part of the box model */
}

.paper-container {
  width: 90%;
  max-width: 600px;
  background-color: #fff;
  box-shadow: 5px 5px 15px rgba(0, 0, 0, 0.5);
  padding: 30px;
  border: 3px solid black;
  border-radius: 10px;
  position: relative;
  margin: auto; /* Magically centers vertically if there's space */
}
```
