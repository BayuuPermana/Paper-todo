# Mobile Neural Hub Implementation Plan

## Overview
Transform the mobile UI into a tabbed dashboard with a sticky tool header.

## Current State Analysis
- `App.jsx`: Stacks Sidebar, Workspace, and Tools vertically.
- `right-segment`: Hidden on scroll or far down the page.

## Implementation Approach
1. **State Addition**: Add `activeTab` to `App.jsx`.
2. **CSS Global Overhaul**: Add fixed positioning rules for header and footer.
3. **CommandHeader Component**: A compact, mobile-only top bar.
4. **BottomMatrix Component**: A mobile-only tab bar.
5. **Conditional Rendering**: Hide segments based on `activeTab` when viewport is small.

## Phase 1: State & CSS (mobile-001, mobile-004)
### Overview
Establish the infrastructure for view switching.

### Changes Required:
#### 1. `src/index.css`
**Changes**: Add mobile-specific visibility and fixed layout classes.
```css
.mobile-only { display: none; }
@media (max-width: 1200px) {
  .mobile-only { display: block; }
  .desktop-only { display: none !important; }
  .app-layout { padding: 60px 0; } /* Space for fixed bars */
  .active-pane { display: block !important; }
  .inactive-pane { display: none !important; }
}
```

#### 2. `src/App.jsx`
**Changes**: Add `activeTab` state and logic to switch tabs when a project is selected.

## Phase 2: UI Components (mobile-002, mobile-003)
### Overview
Build the sticky header and footer.

### Changes Required:
#### 1. `src/App.jsx` (or separate components)
**Changes**: Implement `CommandHeader` and `BottomMatrix` in the main render.

### Success Criteria:
#### Automated:
- [ ] `bun run build`
