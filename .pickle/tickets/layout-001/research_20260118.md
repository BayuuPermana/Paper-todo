# Research: Implement 2/3 - 1/3 Responsive Split Layout

**Date**: 2026-01-18

## 1. Executive Summary
The current application uses a single-column layout centered in the viewport. The `.paper-container` is constrained to 600px. Implementing a dual-pane layout requires broadening this container or wrapping it in a higher-level grid system to accommodate the new Pomodoro and Calendar components.

## 2. Technical Context
- **File**: `src/App.jsx`
    - Renders `AddTodo` and `DragDropContext`/`TodoList` within a single `div.paper-container`.
- **File**: `src/index.css`
    - `body` uses `display: flex; justify-content: center;`.
    - `.paper-container` (Lines 11-20) has `width: 90%; max-width: 600px;`.

## 3. Findings & Analysis
- **Current Layout**: Centered column.
- **Proposed Layout**:
    - A wrapper container (e.g., `.app-layout`) that uses `display: grid; grid-template-columns: 2fr 1fr;`.
    - The left pane will house the existing todo list logic.
    - The right pane will house the upcoming `PomodoroTimer` and `PaperCalendar`.
- **Responsiveness**: The `grid-template-columns` needs to switch to `1fr` (vertical stack) for screens below a certain width (e.g., 1024px).

## 4. Technical Constraints
- **Aesthetic Persistence**: All new layout containers must share the "Paper" visual properties (shadows, borders, font).
- **Vite/React**: Standard React component architecture.

## 5. Architecture Documentation
- Current: Single root component (`App`) managing global state and rendering a single pane.
- Targeted: Root component managing global state and layout, delegating panes to sub-containers.
