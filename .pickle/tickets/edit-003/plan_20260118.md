# Temporal Text Manipulator Implementation Plan

## Overview
Implement inline editing for tasks and subtasks. This involves adding update handlers to the root state and creating editable UI states in the child components.

## Current State Analysis
- `App.jsx`: Missing update handlers for text modification.
- `TodoItem.jsx` / `SubTaskItem.jsx`: Static text display with no editing mode.

## Implementation Approach
1. **Root Logic**: Add `editTodo` and `editSubTask` to `App.jsx`.
2. **UI Toggle**: Add `isEditing` state to `TodoItem` and `SubTaskItem`.
3. **Editable Input**: Use a styled `input` that appears on demand.
4. **Triggers**: Add an "Edit" icon (pencil) to the action areas.

## Phase 1: Root Logic (edit-002)
### Overview
Add the state mutation functions to `App.jsx`.

### Changes Required:
#### 1. `src/App.jsx`
**Changes**: Add `editTodo` and `editSubTask`.
```javascript
  const editTodo = (id, newText) => {
    setTodos(todos.map(todo => 
      todo.id === id ? { ...todo, text: newText } : todo
    ));
  };

  const editSubTask = (todoId, subTaskId, newText) => {
    setTodos(todos.map(todo => {
      if (todo.id === todoId) {
        return {
          ...todo,
          subTasks: todo.subTasks.map(st => 
            st.id === subTaskId ? { ...st, text: newText } : st
          )
        };
      }
      return todo;
    }));
  };
```
