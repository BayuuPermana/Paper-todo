# Research: Integrated Input Matrix Architecture

**Date**: 2026-01-18

## 1. Executive Summary
The current input areas suffer from visual fragmentation. By refactoring the input and buttons into a unified flexbox-driven 'Matrix' bar, we can achieve a cleaner 'Paper' aesthetic while maintaining high-performance interaction. The submission symbol will be upgraded to '»' for better directional clarity.

## 2. Technical Context
- **Components**: `AddTodo.jsx`, `TodoItem.jsx`.
- **CSS File**: `src/index.css`.
- **Existing Elements**: `input[type="text"]`, `button` (add/attach).

## 3. Findings & Analysis
- **Unified Container**: A wrapper `div` with `border: 2px solid black` and `border-radius: 5px` will house the elements.
- **Internal Stripping**: The `input` field must have `border: none` and `background: transparent`.
- **Button Integration**: Buttons will be positioned at the ends of the bar. The `+` (image) on the left and `»` (submit) on the right.
- **Vertical Alignment**: `align-items: center` on the flex wrapper ensures the [+] and [»] stay centered relative to the text.
- **Horizontal Stability**: Since we use `flex-direction: row` without wrapping, the bar will stay horizontal on mobile.

## 4. Technical Constraints
- **Accessibility**: Inputs must still have clear focus states (we'll apply focus-within to the wrapper).
- **Mobile Width**: The sidebar width (280px) is the primary constraint for the task input.

## 5. Architecture Documentation
- Class: `.input-matrix`.
- Sub-classes: `.matrix-btn-left`, `.matrix-btn-right`, `.matrix-input`.
