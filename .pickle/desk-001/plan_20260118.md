# Tauri & SQLite Scaffolding Implementation Plan

## Overview
Initialize the Tauri project, install dependencies, and configure the SQLite plugin.

## Current State Analysis
- Pure web app.
- No `src-tauri` directory.
- `localStorage` usage throughout `App.jsx`.

## Implementation Approach
1. Install `@tauri-apps/cli` and `@tauri-apps/api`.
2. Run `tauri init`.
3. Install `tauri-plugin-sql` Rust dependency.
4. Set up the SQL plugin in `main.rs`.
5. Create a `db.js` utility in the frontend to wrap SQL operations.

## Phase 1: Scaffolding
### Overview
Initialize Tauri.

### Changes Required:
#### 1. Shell Commands
**Action**: Install CLI and init.
```bash
bun add -d @tauri-apps/cli
bun add @tauri-apps/api
bunx tauri init --app-name "Paper Todo" --window-title "Paper Todo" --dist-dir "../dist" --dev-path "http://localhost:5173" --before-dev-command "bun run dev" --before-build-command "bun run build"
```

## Phase 2: Rust Configuration
### Overview
Add SQL plugin to Rust core.

### Changes Required:
#### 1. `src-tauri/Cargo.toml`
**Action**: Add `tauri-plugin-sql`.
```toml
[dependencies]
tauri-plugin-sql = { git = "https://github.com/tauri-apps/plugins-workspace", branch = "v1" }
```

#### 2. `src-tauri/src/main.rs`
**Action**: Register the plugin.
```rust
tauri::Builder::default()
  .plugin(tauri_plugin_sql::Builder::default().build())
  // ...
```

### Success Criteria:
#### Manual:
- [ ] `bunx tauri dev` launches a window.
- [ ] `src-tauri` directory exists and compiles.
- [ ] SQL plugin is registered without panics.
