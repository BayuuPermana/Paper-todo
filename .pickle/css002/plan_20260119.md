# Sidebar Spacing Implementation Plan

## Overview
Ensure sidebar task containers are fully visible and don't overlap with the scrollbar.

## Current State Analysis
- `TaskSidebar.jsx`: Items use `width: '100%'`, causing clipping of shadows/borders in the `overflow-x: hidden` parent.
- `index.css`: `.sidebar-list` padding is insufficient for the shadow + scrollbar combo.

## Implementation Approach
1. Update `index.css`:
    - Set `.sidebar-list` padding to `10px 25px 10px 10px`.
2. Update `TaskSidebar.jsx`:
    - Remove `width: '100%'`.
    - Add `margin: '0 5px'` to ensure the shadow has internal clearance.
    - Set `box-sizing: 'border-box'`.

## Phase 1: CSS & Component Update
### Overview
Adjust padding and margins for spacing.

### Changes Required:
#### 1. `src/index.css`
**Changes**: Update `.sidebar-list` padding.

```css
.sidebar-list {
  /* ... */
  padding: 10px 25px 10px 10px; /* Right padding for scrollbar + shadow */
}
```

#### 2. `src/TaskSidebar.jsx`
**Changes**: Remove `width: '100%'` and add horizontal margin.

```jsx
// inside map
<div
  role="button"
  onClick={() => onSelectTodo(todo.id)}
  style={{
    // width: '100%', REMOVE
    margin: '0 5px',
    // ...
  }}
>
```

### Success Criteria:
#### Manual:
- [ ] Task containers have visible left and right borders.
- [ ] Box shadows are fully visible on the right.
- [ ] Clear gutter between container shadow and scrollbar.
